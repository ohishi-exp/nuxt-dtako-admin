# 拘束時間 → 時間給計算の仕様 (restraint-wage)

/restraint-wage (拘束・賃金) の時間給計算・最低賃金チェックが「何を・どう計算しているか」の
仕様まとめ。実装は `workers/dtako-scraper-relay/src/restraint-wage.ts` (pure ロジック、
coverage 100% gate 対象)、表示は `app/pages/restraint-wage.vue` /
`app/components/RestraintWageMonthlyTable.vue`。

Refs #244 (時間給計算の導入), #253 (給与比較・最低賃金), #268 (タブ分離), #278 (支給項目5区分化),
#282 (時給列廃止・計算単価表示)

## 1. 位置づけ — 理論値の事前チェックであって支払い実績の検証ではない

ここで計算する金額はすべて **「単価マスタ (会社が登録した基本時間単価) × デジタコ拘束時間」の
理論値**。実際に振り込まれた給与 (支払い実績) ではない。

- 目的: **単価マスタの設定自体が最低賃金水準を満たしているか**を、同じ式で計算した
  「単価マスタ換算」と「最低賃金換算」の理論値同士で比較する (レート設定の事前チェック)。
- 支払い済み金額の検証は「給与比較」タブ (`app/utils/salary-compare.ts`、給与明細 CSV との突合) が担当。

## 2. 時間データの出どころ

| 値 | 出どころ |
|---|---|
| 実働・時間外・深夜・時間外深夜 (日別) | theearth (web地球号) CSV の日別値をそのまま採用 |
| 法定時間内 | 自前導出: **実働 − 時間外 − 時間外深夜** |
| 休日区分 (法定/法定外) | 自前分類 (曜日ベース、下記 §4) |
| 週40h 超過 | 自前計算 (§5) |

休日 (isRestDay) の日は全時間 0 として扱う。

## 3. 法定区分と係数 (DEFAULT_WAGE_CONFIG)

金額 = 区分時間(h) × 基本時間単価 × 係数。**円未満は区分毎に四捨五入**し、合計は丸め後の総和。
係数は wage-config (R2 JSON) で変更可能。既定値:

| 区分 (key) | 係数 | 備考 |
|---|---|---|
| 法定時間内 (statutory) | 1.0 | 実働 − 時間外 − 時間外深夜 |
| 法定時間外 (overtime) | 1.25 | theearth 日別値 |
| 深夜 (night) | **0.25 (加算分のみ)** | §4 参照。基礎 1.0 は statutory 側に含まれる |
| 時間外深夜 (overtimeNight) | 1.5 | 1.25 + 0.25 の合成 |
| 法定外休日 (nonLegalHoliday) | 1.25 | |
| 法定外休日深夜 (nonLegalHolidayNight) | 1.5 | |
| 法定休日 (legalHoliday) | 1.35 | |
| 法定休日深夜 (legalHolidayNight) | 1.6 | |
| 週40超過 (weekly40Excess) | 1.25 | |
| 月60h超過 (overtimeOver60h) | 1.5 | 最低賃金ベース残業代のみ (§6) |

## 4. 日別分類ルール

- **平日**: 法定内 = 実働 − 時間外 − 時間外深夜。深夜は 0.25 加算対象の分数として別枠計上。
- **法定休日 (既定 日曜、wage-config で変更可)**: 時間外の概念なし — 実働すべてを法定休日
  (深夜分は法定休日深夜) に計上。
- **法定外休日 (既定 土曜)**: 実働すべてを法定外休日 (深夜分は法定外休日深夜) に計上。
  会社カレンダー (祝日・所定休日の個別指定) には未対応の近似。

### 深夜 (通常) は「0.25 加算分のみ」の別枠

深夜時間帯の基礎 1.0 は法定時間内 (statutory) に含まれており、深夜 (night) 区分は
**0.25 倍の上乗せ分だけ**を持つ。したがって恒等式:

```
合計 = 実働全体 × 基礎単価
     + 割増分 (時間外 0.25 / 時間外深夜 0.5 / 深夜 0.25 / 休日系の超過分)
```

が成り立ち、基礎の二重計上はない。UI の深夜 (通常) 列の計算単価 (@) が基礎単価の
約 0.25 倍になるのはこのため。

## 5. 週40h 超過 (weekly40Excess)

- 週の起算は **日曜** (weekStartsOn=0、決定事項 Refs #244)。
- **週はその終端 (起算+6日) が属する月に計上**する。月初の跨ぎ週は前月末日の実働を含めて
  計算する (前月 summary の days を渡す)。
- 週合計から**法定休日の実働は除外** (法定休日は 1.35 倍側で独立して計上済み)。
- 二重計上防止: 週内で既に割増計上済みの分 (時間外・時間外深夜・法定外休日の実働) を
  40h 超過分から控除し、正の範囲のみ計上する。

```
週40超過 = max(0, 週実働合計(法定休日除く) − 40h − 週内の割増計上済み時間)
```

## 6. 月60h 超過の時間外割増 (最低賃金ベース残業代)

改正労基法 (中小企業も 2023-04〜適用) の月60h 超 1.5 倍は、**最低賃金換算の残業代理論値**
(minWageOvertimePay / minWageNightOvertimePay) に適用している。

- 60h 判定の対象 = **時間外 + 時間外深夜 + 週40超過の合算**。
- 時間外軸 (60h まで 1.25、超過分 1.5) と深夜軸 (常時 +0.25) は独立加算。
  60h 超の深夜残業は 1.5 + 0.25 = **1.75 倍相当** (単純な係数切替では表せない)。
- 表示上は 60h 枠を「通常残業 (時間外+週40超過)」から先に消費する按分で 2 列
  (残業代 / 深夜残業代) に割り振るが、これは表示上の慣行であり、按分順序を変えても
  2 列合計は不変 (`splitMinWageOvertimePay` の doc comment 参照)。
- 単価マスタ換算側 (actualOvertimePay 等) は係数一定 (1.25 / 1.5) で、60h 切替は入れていない。

## 7. 最低賃金チェックの比較設計 — 何を含み、何を含まないか

最低賃金法4条3項により、**割増賃金 (時間外・休日・深夜) は最低賃金の対象賃金から除外**して
比較するのが法定の計算。このため:

- 各列の比較は**同じ区分・同じ式の理論値同士** (単価マスタ換算 vs 最低賃金換算) で行う。
  法的な意味で最低賃金比較になっているのは **基本給(法定内) 列**
  (基礎単価 × statutory 時間 vs 最低賃金 × statutory 時間 — 実質的に基礎単価 ≥ 最低賃金の判定)。
- 他の列 (深夜・残業代・合計) は「最低賃金を基礎額とした場合の理論値」との差分表示で、
  単価マスタの水準感を区分毎に見るための参考値。
- **廃止**: 「換算時給 (時間給合計 ÷ 実働) vs 最低賃金」の比較列 (時給列) は、分子に割増が
  全部乗った時給を割増除外前提の法定下限と比べており意味がないため削除した (Refs #282)。
  「最低賃金割れ」バナーは合計(計算) の差 (totalPayDiff) 基準。
- 計算単価 (@) は 金額 ÷ 対象時間 の実額按分 (円/h)。基本給の @ は基礎単価そのもの、
  深夜(通常) の @ は加算分 0.25 倍のみの単価。

### 給与実績側 (給与比較タブ) の支給項目区分 (Refs #278)

給与明細 CSV の支給項目は「割増賃金の基礎 (労基法37条) × 最低賃金の対象 (最低賃金法4条3項)」
の 2 軸で 5 区分に分類する (`SalaryItemCategory`):

| 区分 | 割増基礎 | 最低賃金対象 | 例 |
|---|---|---|---|
| base | ○ | ○ | 基本給・職務手当 |
| overtime | — (割増そのもの) | × | 残業・深夜・休日出勤手当 |
| minwage-only | × | ○ | 住宅・別居・子女教育手当 |
| premium-base-only | ○ | × | 精皆勤手当 |
| excluded | × | × | 通勤・家族手当、臨時の賃金、賞与 |
